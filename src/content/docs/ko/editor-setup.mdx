코드 에디터를 커스터마이즈 하여 Astro 개발자 환경을 개선하고 새로운 기능을 잠금 해제하세요.

## VS Code

VS Code는 Microsoft에서 웹 개발자를 위해 만든 인기있는 코드 에디터입니다. VS Code는 [GitHub 코드스페이스](https://github.com/features/codespaces) 및 [Gitpod](https://gitpod.io/)와 같은 인기있는 브라우저 내 코드 에디터에도 탑재되어 있습니다.

Astro는 모든 코드 에디터에서 작동합니다. 그러나 VS Code는 Astro 프로젝트에 권장되는 에디터입니다. Astro 프로젝트에 대한 몇 가지 주요 기능과 개발자 환경 개선을 제공하는 [공식 Astro VS Code 확장 프로그램](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode)을 유지 관리하고 있습니다.

이 확장 프로그램은 다음과 같은 기능을 포함합니다.

- `.astro` 파일에 대한 구문 강조 표시.
- `.astro` 파일에 대한 TypeScript 타입 정보.
- 코드 자동완성, 힌트 등을 위한 [VS Code Intellisense](https://code.visualstudio.com/docs/editor/intellisense)

시작하려면 지금 바로 [Astro VS Code 확장 프로그램](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode)을 설치하세요.

📚 Astro 프로젝트에서 [타입스크립트 설정하는 법](/ko/guides/typescript/)을 참조하세요.

## JetBrains IDEs

Astro에 대한 초기 지원은 WebStorm 2023.1에서 시작되었습니다. 공식 플러그인은 [JetBrains Marketplace](https://plugins.jetbrains.com/plugin/20959-astro)를 통해 설치하거나 IDE의 플러그인 탭에서 "Astro"를 검색하여 설치할 수 있습니다. 이 플러그인에는 구문 강조 표시, 코드 완성 및 포멧팅과 같은 기능이 포함되어 있으며 향후 더 많은 고급 기능을 추가할 계획입니다. 이 플러그인은 [다른 모든 JavaScript가 지원되는 JetBrains IDE](https://www.jetbrains.com/products/#lang=js&type=ide)에서도 사용할 수 있습니다.

곧 출시될 JetBrains의 [Fleet IDE](https://www.jetbrains.com/fleet/)에서는 언어 서버 지원이 포함되며 공식 Astro 툴링을 사용할 수 있습니다.

## 다른 코드 에디터

저희의 멋진 커뮤니티에서 다른 인기있는 에디터들을 위해 다음과 같은 여러가지 확장 기능을 관리하고 있습니다.

- [Open VSX 에 있는 VS Code Extension](https://open-vsx.org/extension/astro-build/astro-vscode) <span style="margin: 0.25em;"><Badge variant="accent">공식</Badge></span> - 공식 Astro VS Code 확장은 [VSCodium](https://vscodium.com/)과 같은 개방형 플랫폼용 Open VSX 레지스트리에서 사용할 수 있습니다.
- [Nova Extension](https://extensions.panic.com/extensions/sciencefidelity/sciencefidelity.astro/) <span style="margin: 0.25em;"><Badge variant="neutral">커뮤니티</Badge></span> - 노바 내에서 Astro에 구문 강조 표시 및 코드 완성 기능을 제공합니다.
- [Vim Plugin](https://github.com/wuelnerdotexe/vim-astro) <span style="margin: 0.25em;"><Badge variant="neutral">커뮤니티</Badge></span> - Vim 또는 Neovim 내에서 Astro에 구문 강조 표시, 들여쓰기 및 코드 접기 지원을 제공합니다.
- Neovim [LSP](https://github.com/neovim/nvim-lspconfig/blob/master/doc/server_configurations.md#astro)와 [TreeSitter](https://github.com/virchau13/tree-sitter-astro) Plugins <span style="margin: 0.25em;"><Badge variant="neutral">커뮤니티</Badge></span> - Neovim 내에서 Astro를 위한 구문 강조 표시, 트리시터 구문 분석 및 코드 완성 기능을 제공합니다.

## 웹 에디터

로컬 에디터 외에도 Astro는 다음과 같은 웹 에디터들에서도 잘 실행됩니다.

- [StackBlitz](https://stackblitz.com/)와 [CodeSandbox](https://codesandbox.io/)는 브라우저에서 실행되는 온라인 에디터로 `.astro` 파일에 대한 구문 강조 표시 기능이 기본으로 지원됩니다. 설치나 구성이 필요 없습니다!
- [GitHub.dev](https://github.dev/)를 사용하면 [웹 확장](https://code.visualstudio.com/api/extension-guides/web-extensions)으로 Astro VS Code 확장 프로그램을 설치할 수 있습니다. 확장 기능 전체를 지원하지만 현재는 구문 강조 표시 기능만 지원됩니다.
- [Gitpod](https://gitpod.io/)는 Open VSX의 공식 Astro VS Code 확장을 설치할 수 있는 클라우드 개발 환경입니다.

## 다른 도구들

### ESLint

[ESLint](https://eslint.org/)는 자바스크립트와 JSX에 널리 사용되는 린터입니다. Astro 지원을 위해 [커뮤니티 유지 관리 플러그인](https://github.com/ota-meshi/eslint-plugin-astro)을 설치할 수 있습니다.

프로젝트에 ESLint를 설치하고 설정하는 방법에 대한 자세한 내용은 [프로젝트 사용자 가이드](https://ota-meshi.github.io/eslint-plugin-astro/user-guide/)를 참조하세요.

### Prettier

[Prettier](https://prettier.io/)는 JavaScript, HTML, CSS 등에 널리 사용되는 포맷터입니다. [Astro VS Code 확장](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode) 또는 [다른 에디터 내의 Astro 언어 서버](#다른-코드-에디터)를 사용하는 경우 Prettier를 사용한 코드 포멧팅이 기본으로 포함되어 있습니다.

에디터 외부(예: CLI) 또는 확장 프로그램을 지원하지 않는 에디터 내부에서 `.astro` 파일 포멧팅에 대한 지원을 추가하려면 [공식 Astro Prettier 플러그인](https://github.com/withastro/prettier-plugin-astro)을 설치하세요.

시작하려면 먼저 Prettier와 플러그인을 설치하세요.

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm install --save-dev prettier prettier-plugin-astro
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm install -D prettier prettier-plugin-astro
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn add --dev prettier prettier-plugin-astro
  ```
  </Fragment>
</PackageManagerTabs>

그러면 Prettier가 플러그인을 자동으로 감지하여 실행할 때 `.astro` 파일을 처리하는 데 사용합니다.

```shell
prettier --write .
```

지원되는 옵션, VS Code 내에서 Prettier를 설정하는 방법 등에 대한 자세한 내용은 [Prettier 플러그인 README](https://github.com/withastro/prettier-plugin-astro/blob/main/README.md)를 참조하세요.

:::caution[pnpm과 함께 사용]
Prettier 내부의 업스트림 이슈로 인해 [pnpm](https://pnpm.io/) 사용 시 플러그인이 자동으로 감지되지 않습니다. 이를 해결하려면 Prettier 실행 시 다음 파라미터를 추가해야 합니다.

```shell
prettier --write . --plugin=prettier-plugin-astro
```

VS Code 내에서 Prettier를 사용할 때도 추가 설정이 필요합니다. 자세한 내용은 플러그인의 README를 참조하세요.
:::
